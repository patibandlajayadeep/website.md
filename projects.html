Healthcare & Order-Driven Projects

Real examples of how I’ve improved claims accuracy, reduced manual work, and made EHR workflows easier for billing staff and SMEs.

Claims & Pricing

Claims Rules Validation Studio

A web application used by analysts to configure and test rules like ESRD pricing, bundling logic, and clinical edits before they are pushed into production.

  • Built API-driven rule test harness in .NET / Web API with SQL Server backend.
  • Created UI where analysts can run sample claims, inspect each rule fire, and export audit logs for compliance.
  • Reduced production rule defects by 30%+ by catching issues earlier.
Orders & EHR Workflow

Order Journey Explorer

A timeline view for lab, imaging, and procedure orders that shows each step – from placement to billing – with warnings when something is stuck.

  • Designed microservice APIs to track order events and statuses.
  • Front-end built with Blazor / React and animated transitions so staff can see delays instantly.
  • Helped operations team reduce average order turnaround time by several hours.
Vendor Apps

Vendor Configuration Portal

A secure internal web application that allows business teams to configure payers, fee schedules, vendors, reimbursement settings, and workflow rules—all without requiring direct database access.

  • Implemented role-based access and fine-grained auditing with .NET Core and Azure AD.
  • Implemented an approval workflow for configuration changes with reviewers, email notifications, and full audit history.
  • Added detailed auditing and activity logs, improving compliance and traceability for financial and claim-processing changes.
  • Reduced manual DBA “DB change” tickets by more than half, speeding up releases.
  • Improved data integrity by validating configuration changes before they reach production.
